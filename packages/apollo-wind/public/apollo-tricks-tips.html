Tricks & Tips Open Storybook in Claude Code
Prerequisites
  • Claude Code desktop app installed and signed in
  • The apollo-ui repo cloned locally
Steps
1
Open the project in Claude Code
From the terminal, cd into the repo and launch Claude Code:
cd apollo-ui claude
Or open the desktop app and select the apollo-ui folder.
2
Install dependencies (first time only, or after a fresh clone)
Type in the Claude chat:
pnpm install
3
Build the packages (first time only)
pnpm build
4
Ask Claude to open Storybook in the preview
Type exactly:
Open Storybook in the preview panel
How it works: The .claude/launch.json file tells Claude Code how to start the Storybook server and which port to use (6007). Once it's there, you only need to ask Claude to open it — no manual server startup needed.

Note: After the first run, Claude Code caches the server. On subsequent sessions just ask "Open Storybook in the preview" and it'll be near-instant.

Tricks & Tips 5 more tips for designers and PMs
Tip 1Just describe what you want — you don't need to write code
With this Claude + Storybook setup, describe changes in plain English and Claude will make them for you. No React or TypeScript needed.
"Add a node called 'Approval step' between Validate input and Transform data"
"Make the flow show a failed state on the third node"
"Change the layout so the left panel is open by default"
Tip 2Use Storybook as your source of truth for what's possible
Before asking for something custom, browse the sidebar first. Everything listed is a real, built component — not a mockup. If it exists in Storybook, it's production-ready and available to engineers right now.
Tip 3Create a new story page for each concept you're exploring
Ask Claude to create a new page under your section for each idea. This keeps experiments separate and lets you share specific URLs with teammates.

localhost:6007/?path=/story/canvas-templates-flow--my-experiment
Tip 4Use the theme toggle to check both light and dark
The Future Dark / Light toggle in the top toolbar switches the entire preview between themes. Always check your layout in both before handing off — some node states and colors are theme-specific and can look very different.
Tip 5Prototypes generated in Storybook represent the real engineering UI code
What you see in the Storybook preview is rendered with real production components, real tokens, and real spacing — not Figma approximations. Screenshots from here are accurate enough to use in PRDs, design reviews, and presentations.

Apollo Storybook now uses a new domain. Apollo Wind and Apollo React have been merged into a single Storybook, so you can build with both frontend UI and Canvas components in one place.
