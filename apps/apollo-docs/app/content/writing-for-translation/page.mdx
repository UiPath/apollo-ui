# Writing for translation

Write content that translates clearly and accurately across languages. Some languages (e.g. German, Finnish) expand text by 20–30%, so short, literal phrasing is essential.

---

## Guidelines

### Avoid idioms and colloquialisms

Use literal language that translates directly.

| Instead of | Use |
|---|---|
| Hit the ground running | Start immediately |
| On the same page | In agreement |
| Circle back | Follow up |

### Avoid compound nouns

Break compound nouns into separate words or use prepositions. They're harder to translate and often become ambiguous.

| Instead of | Use |
|---|---|
| Data upload process | Process for uploading data |
| User permission settings | Settings for user permissions |

### Use consistent terminology

Don't alternate between synonyms unless the terms mean different things. Pick one and use it throughout.

<Callout type="warning">
  Don't switch between "save" and "store," or "delete" and "remove," unless the distinction is intentional.
</Callout>

### Keep sentences short

Aim for one idea per sentence. Short sentences are easier to translate and easier to read.

### Avoid ambiguous pronouns

Be explicit about what "it," "this," or "that" refers to.

| Instead of | Use |
|---|---|
| It will be deleted. | The file will be deleted. |
| This will take a moment. | Saving will take a moment. |

### Avoid cultural references

Use universal concepts rather than region-specific examples, holidays, or idioms.

### Avoid phrasal verbs

Use single-word verbs when possible. Phrasal verbs are difficult to translate accurately.

| Instead of | Use |
|---|---|
| Set up | Configure |
| Fill out | Complete |
| Look into | Investigate |

**Exception:** Established UI conventions like "sign in" and "log out" are fine to keep.
