# Navigation types

Navigation types define the primary ways users move through the application. This page documents all navigation patterns in use, with guidance on how and when to apply each type.

## Global navigation

Gives users access to key products or product areas across the entire platform.

**Elements:** Global header, waffle menu, global search, notifications, help, profile and preferences

**Best practices:**
- Include only the most critical items to avoid clutter
- Keep the global navigation always accessible
- Use clear, concise, and familiar labels — avoid jargon or internal terms
- Implement consistently across the platform

## Service-level navigation

Lets users navigate within a specific product on the platform.

**Elements:** Panels, secondary and tertiary sidebar menus, tabs, breadcrumbs

**Best practices:**
- Design to reflect the product's hierarchy
- Only show items related to the specific product
- Group items by function when creating dropdowns or tabs
- Keep labels brief but descriptive

## Contextual navigation

Provides users with links to content related to the task they are currently working on.

**Elements:** Hyperlinks, buttons, dropdowns, menus

**Best practices:**
- Use action-oriented language that communicates what will happen when selected
- Be as specific as possible (e.g., "Learn about AI units" not "Learn more")
- Avoid overwhelming the interface with excessive contextual links — include only what is relevant

## Search-based navigation

Allows users to find information or features by searching for specific terms.

**Elements:** Global search, local search, filters, sort options

**Best practices:**
- Keep the search bar easily visible and accessible
- Ensure filter and sort labels reflect users' mental model

## Pagination and scrolling

Helps users navigate through content that is divided into pages or displayed vertically.

**Elements:** Pagination controls, scrolling

**Best practices:**
- Use numbered pagination with clear forward and backward labels
- Clearly indicate which page the user is currently on

## Footer navigation

Provides access to secondary or less frequently used content.

**Elements:** Footer links

**Best practices:**
- Reserve for secondary or less critical content to keep the main navigation clean
- Group related links under meaningful category labels
- Keep labels clear and descriptive — avoid legalese or technical terms
