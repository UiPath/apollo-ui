# Application layouts

UiPath products use two primary application layouts: **Canvas** and **Non-canvas**.

## Layout types

| Layout | Used by |
|---|---|
| **Canvas** | Studio, Data Fabric, Agent Builder, Maestro |
| **Non-canvas** | Test Manager, Orchestrator, Admin, most landing pages |

### Canvas layout

The canvas layout is designed for workspace-based tools where users build, view, and edit flows or content in a visual editing environment. It typically uses panels on the left and right sides of a central editing area.

### Non-canvas layout

The non-canvas layout is used for data-driven and administrative experiences. It typically follows a standard page structure with a global header, side navigation, and a main content area.
