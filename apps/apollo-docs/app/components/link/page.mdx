# Link

Links are used as navigational elements. They may appear on their own, within a sentence or paragraph, or directly following the content.

## Usage

Links are used as navigational elements and can be used on their own or inline with text. They provide a lightweight option for navigation.

Use links when you want users to:

- Navigate to a different page within the application
- Navigate to an entirely different site
- Jump to an element on the same page

## Variants

| Variant | Description |
|---|---|
| **Inline** | Appears within a sentence or paragraph, matching the surrounding text size |
| **Standalone** | Appears on its own, not embedded in body text |

## States

Link has five states: default, hover, press, visited, and focus. Both inline and standalone links exhibit the same behavior.

## Opening links in a new tab

Do not open links in a new tab by default. Desktop users are trained to right-click to open in a new tab.

**Exception:** Open in a new tab if the URL takes the user to an external site.

## Guidelines

- When inline, link size should match the type size of the text in that line or paragraph
- Standalone links should use a text size complementary to the nearby content
- Do not mix the usage of links and buttons — links are for navigation, buttons are for actions
- Make all link text unique and descriptive — avoid "click here" or "link"
- Avoid using the word "link" in the link text, as screen readers will announce it automatically

## Accessibility

- Each link should clearly describe its purpose (WCAG 2.4.4)
- Links to the same destination should use consistent link text across pages (WCAG 3.2.4)
