# Chip

Chips are compact components that display discrete information.

## Usage

Chips allow users to enter information, make selections, filter content, or trigger actions.

Use chips in these instances:

- To add filters
- To select users and groups

| Variant | Purpose |
|---|---|
| **Choice chips** | A compact alternative to radio buttons. Containing at least two options, choice chips represent a single selection. |
| **Filter chips** | A compact alternative to checkboxes. Allow users to select multiple options and filter content for a collection. |
| **Input chips** | Represent discrete pieces of information entered by a user within an input field. Enable user input and verify that input by converting text into chips. |

## Anatomy

The anatomy of a chip is based on what function it is serving.

**Input chip elements:**
- Container
- Text label / chip value
- Chip icon (optional)
- Remove icon (optional, input and filter chips only)

**Chip input container elements:**
- Field label
- Container

## Size and spacing

There are two sizes for chips: **high-density** and **low-density**.

The height of the chip depends on how dense the design is. Text labels should be concise. The width of the chip is variable — once a chip has reached its character limit, it should be truncated with an ellipsis (…).
