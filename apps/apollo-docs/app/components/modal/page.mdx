# Modal

Modals focus the user's attention exclusively on one task or piece of information via a window that sits on top of the page content.

## Usage

Modals are a type of dialog used for interactions between the user and the system. They are designed to present critical information or request input required to complete a workflow.

Use modals sparingly for short, non-frequent tasks such as editing or managing content. They temporarily interrupt the user's workflow.

## Anatomy

The modal has three distinct zones: a header, a body, and a footer. Components such as a data table, form, or progress indicator can occupy the full width of the modal.

| Element | Description |
|---|---|
| **Header** | Includes a title, optional label, and the close icon |
| **Body** | Contains the information and controls needed to complete the modal's task |
| **Footer** | Contains the main actions needed to complete or cancel the task |
| **Close (×)** | Closes the modal without data submission |
| **Overlay** | Screen overlay that obscures the on-page content |

## Sizes

| Size | Description |
|---|---|
| **Small** | For simple confirmations or brief content |
| **Medium** | For standard tasks with moderate content |
| **Large** | For complex tasks with extensive content |

## Types

| Type | Description |
|---|---|
| **Alert Confirm** | For critical confirmations requiring user action |
| **Alert Error** | Communicates a system error |
| **Alert Info** | Provides informational context |
| **Alert Warning** | Warns about potential consequences |
