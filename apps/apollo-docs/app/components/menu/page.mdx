# Menu

A menu is an interactive list of options that a user can choose. The options are hidden by default and revealed when a user interacts with an element. Menus are often used to show actions or navigation. They're different from Select because they're not form inputs.

## Usage

Menus are always complete (no overflow) and exactly as long as needed to fit all items. Menu items have default, hover, and selected states.

- Menus are strongly encouraged to have only one flyout level (two levels total)
- Menus can be displayed on top of another surface
- Only one menu can be active on a screen at any time

## Anatomy

| Element | Description |
|---|---|
| **Section header** | (Optional) Groups related menu items |
| **Menu container** | The boundary of the menu |
| **Menu item** | An individual selectable option |
| **Icon** | (Optional) Leading icon for a menu item |
| **Line divider** | (Optional) Visual separator between groups |
| **Secondary label** | (Optional) Additional context for a menu item |
| **Container elevation** | Shadow applied to the menu container |

## Appearance

### Size and spacing

The recommended minimum width for a menu is 192px.

### Leading and trailing content

Menu items can have leading content (icons, checkboxes, or radio buttons) and trailing content.
