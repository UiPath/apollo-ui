# Badge

A badge is a compact label, generally appearing inside or in close proximity to another larger interface component, representing a status, property, or some other metadata.

## Usage

A badge is a non-interactive visual indicator used to highlight an item's status for quick recognition. Badges can be displayed in groups or individually, and express meaning through color and text.

Badges are either small or large and use color to indicate meanings that users can learn and recognize across products.

## Anatomy

| Element | Description |
|---|---|
| **Badge container** | Primary element, color-coded to communicate status |
| **Label** | Short descriptive text describing the badge |

## Appearance

### Status badges

Status badges and their color help users quickly find an object's status or priority.

| Status | Usage |
|---|---|
| **Informative** | Conveys informative messages (e.g., *In progress*) or medium priority |
| **Success** | Positive feedback or success messaging (e.g., *Success*) or low priority |
| **Warning** | Warns users that they need to take notice (e.g., *Warning*) or high priority |
| **Error** | Negative feedback or error messaging (e.g., *Failed*) or critical priority |

### Size

Badges can be displayed in **large** and **small** sizes.

## Guidelines

- Labels should be written in sentence case and kept short to avoid wrapping — one to two words maximum
- The label should clearly convey the status
- Badges are always static and non-interactive
- The colors are tied to meaning in the Apollo Design System — don't change colors arbitrarily
- Do not include icons in badges
