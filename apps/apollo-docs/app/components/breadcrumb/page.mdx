# Breadcrumb

Breadcrumbs are a secondary navigation system that shows a user's location on a site or app. They help users understand where they are relative to other pages and provide convenient links to navigate to different pages.

## Usage

Breadcrumbs are effective in products and experiences that have a large amount of content organized in a hierarchy of more than two levels.

Breadcrumbs are always treated as secondary and should never entirely replace primary navigation.

Use breadcrumbs:

- When the system has more than two layers in a hierarchy
- When you need to inform the user of where they are
- When the user may need to navigate back to a higher level

Place breadcrumbs in the top left corner, typically above the page title.

## Anatomy

| Element | Description |
|---|---|
| **Page link** | Directs users to the parent-level page |
| **Separator** | Clearly distinguishes between each page in the trail |

## States

Breadcrumb items support four states: enabled, hover, active, and focus.

## Overflow content

When space becomes limited, use an overflow menu to truncate the breadcrumbs. The first and last two page links should be shown; remaining breadcrumbs are condensed into an overflow menu. Breadcrumbs should never wrap onto a second line.
