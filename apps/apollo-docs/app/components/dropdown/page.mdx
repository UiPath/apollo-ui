# Dropdown

Dropdown allows a user to select a value from a series of options.

## Usage

A dropdown allows users to select one or more options from a list. It's useful in forms for custom-styled options or in user interfaces to filter and sort data.

Dropdowns can be used in forms, full pages, dialogs, or side panels.

If you need to let someone type in the field to submit a free-form answer or filter the list, use a [Combobox](/components/combobox) instead.

## Anatomy

Dropdowns are composed of four distinct sections: assistive text (labels or helper text), a field, a menu, and options within the menu.

| Element | Description |
|---|---|
| **Label** | Text that informs the user what to expect in the list of dropdown options |
| **Dropdown selector** | Persists when the dropdown is open or closed |
| **Value** | The default placeholder text "Select" is used when there is no default selection |
| **Helper text** | (Optional) Assistive text to help the user choose the right selection |
| **Chevron icon** | Indicates the dropdown can be opened |

## Appearance

### Labeling

The label is positioned outside and above the input field, accompanied by an asterisk (*) if user input is required.

### Placeholder text

Use placeholder text as an extra hint to help people understand the options. If no default selection is shown, use the default placeholder text "Select." Don't use placeholder text to label the dropdown.

## Behaviors

### Single select

Selecting a closed field opens the dropdown menu. Users can only select one option. Choosing an option closes the field and the selected text replaces the placeholder.

### Multi-select

For interactions involving multiple selections, each list item is accompanied by a checkbox. The list stays open until dismissed (by clicking outside or pressing Esc). The selected text replaces the placeholder unless the selections exceed the container width, in which case the field shows the count of selected options.

## Accessibility

- Labels should be clear, concise, and describe the purpose of the dropdown (WCAG 1.3.1)
- Use short, unique names for all options — no two options should start with the same word or phrase (WCAG 1.3.1, 3.3.2)
- Dropdown must be fully keyboard navigable ([W3C Combobox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/))
