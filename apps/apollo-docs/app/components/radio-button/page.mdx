# Radio button

Radio buttons allow users to select exactly one option from a set of mutually exclusive choices.

## Usage

Use radio buttons when users must choose only one option from a list. Unlike checkboxes, radio buttons do not support multiple selections. When a user selects a new option, the previously selected option is automatically deselected.

## Anatomy

| Element | Description |
|---|---|
| **Circle input** | The circular selection indicator |
| **Label** | Text describing the option, aligned to the right of the circle |

## Orientation

Radio buttons can be arranged vertically or horizontally depending on the layout and the number of options.

- **Vertical**: Preferred when options have longer labels or when space allows for scanning clarity
- **Horizontal**: Suitable for short labels with a small number of options

## States

| State | Description |
|---|---|
| **Unselected** | Default state; option not chosen |
| **Selected** | Active state; option is chosen |
| **Unselected disabled** | Option is unavailable and not selected |
| **Selected disabled** | Option is unavailable but was previously selected |

## Guidelines

- Always present radio buttons in a logical order (e.g., most likely to least likely, or alphabetical)
- Always provide a visible label for each radio button
- Use radio buttons only for mutually exclusive selections — use checkboxes for multiple selections
- Avoid pre-selecting options unless a default is clearly appropriate
