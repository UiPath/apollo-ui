# Tabs

Tabs organize related content into alternate views within the same page context.

## Usage

Use tabs to allow users to navigate between different views of the same content or to switch between related sections without leaving the page. At least two tabs must be present, and one tab must always be active.

## Variants

| Variant | Height | Best for |
|---|---|---|
| **Primary** | 40px | Main content areas and top-level navigation within a page |
| **Secondary** | 32px | Nested content or compact layouts |

## States

| State | Description |
|---|---|
| **Enabled** | Tab is available and not selected |
| **Hovered** | Cursor is over the tab |
| **Active** | Tab is currently selected |
| **Focus** | Tab has keyboard focus |
| **Disabled** | Tab is present but not interactive |

## Sizing

Tabs use auto-width by default, sizing to their label content.

## Guidelines

- Order tabs by relevance — most commonly used first
- Sequence tabs by logical association when applicable
- Keep tab labels short: 1–2 words is ideal
- Always keep at least one tab active
- Avoid using tabs for steps in a sequential process — use a stepper instead
