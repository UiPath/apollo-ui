# AI Chat

A reusable AI chat UI component for the Apollo Design System. Built with React, TypeScript, Tailwind CSS, and Zod. Designed to work with [TanStack AI](https://tanstack.com/ai) — you bring `useChat` and a connection adapter, the component handles rendering messages, tool calls, markdown, tool groups, suggestions, and errors.

<div className="not-prose my-8">
  <AiChatTemplate />
</div>

## Features

- **TanStack AI Integration** — Works with `useChat` from `@tanstack/ai-react` and `UIMessage` types
- **Typed Display Tools** — Define display tools with `toolRenderer()` — name and schema come from the TanStack AI tool definition, render args are fully typed
- **AgentHub Adapter** — Built-in custom adapter for the UiPath AgentHub normalized LLM endpoint (OpenAI + Anthropic models)
- **Markdown Rendering** — Renders assistant responses with GitHub Flavored Markdown
- **Collapsible Tool Groups** — Consecutive tool-only steps are grouped into expandable badges
- **Suggestion Buttons** — Interactive choice buttons rendered from tool results
- **Error Display** — Inline error banner for API and network errors
- **i18n Support** — Built-in internationalization via react-i18next
- **Accessible** — WCAG 2.1 compliant with keyboard navigation and ARIA live regions

## Quick Start

```tsx
import { toolDefinition } from '@tanstack/ai';
import { clientTools } from '@tanstack/ai-client';
import { stream, useChat } from '@tanstack/ai-react';
import { AiChat } from '@/components/ui/ai-chat/components/ai-chat';
import { createAgentHubStreamFactory } from '@/components/ui/ai-chat/adapters/agenthub/adapter';

function BasicChat() {
  const config = {
    baseUrl: 'https://cloud.uipath.com/{org}/{tenant}/agenthub_/llm/api',
    model: { vendor: 'openai' as const, name: 'gpt-4o' },
    accessToken: () => getAccessToken(),
    systemPrompt: 'You are a helpful assistant.',
  };

  const connection = stream((messages) =>
    createAgentHubStreamFactory(config)(messages),
  );

  const { messages, sendMessage, isLoading, stop, clear, error } = useChat({
    connection,
  });

  return (
    <AiChat
      messages={messages}
      isLoading={isLoading}
      onSendMessage={(text) => sendMessage(text)}
      onStop={stop}
      onClearChat={clear}
      error={error}
      title="AI Assistant"
    />
  );
}
```

## AgentHub Adapter

The built-in adapter for the **UiPath AgentHub** normalized LLM endpoint. It converts TanStack AI `UIMessage` arrays to the AgentHub wire format, calls the endpoint, and parses the SSE response back into AG-UI `StreamChunk` events.

```tsx
import { createAgentHubStreamFactory, type AgentHubAdapterConfig } from '@/components/ui/ai-chat/adapters/agenthub/adapter';

const config: AgentHubAdapterConfig = {
  baseUrl: 'https://cloud.uipath.com/{org}/{tenant}/agenthub_/llm/api',
  model: { vendor: 'openai', name: 'gpt-4o' },
  accessToken: () => getAccessToken(),
  systemPrompt: 'You are a helpful assistant.',  // also accepts () => string
  maxTokens: 2048,
  temperature: 0.7,
  tools: toolDefs,  // client tools — wire format is derived automatically
};

const connection = stream((messages) =>
  createAgentHubStreamFactory(config)(messages),
);
```

The `model.vendor` field controls wire-format differences:
- **`"openai"`** — flat tool definitions (`{ name, description, parameters }`)
- **`"anthropic"`** — Anthropic tool format (`{ type: "custom", input_schema }`), non-empty assistant content on tool-call messages
- The `X-UiPath-LlmGateway-NormalizedApi-ModelName` header is always sent for routing
- Responses are always OpenAI-compatible SSE regardless of the underlying model

## Display Tools

Display tools render UI inline in the chat when the LLM calls them. Use `toolRenderer()` to create a typed renderer directly from a TanStack AI tool definition — the name and schema come from the definition, so there's no duplication.

```tsx
import { z } from 'zod';
import { toolDefinition } from '@tanstack/ai';
import { clientTools } from '@tanstack/ai-client';
import { buildToolRenderers, toolRenderer } from '@/components/ui/ai-chat/types';

// 1. Define the schema
const showResultsInput = z.object({
  entityName: z.string(),
  columns: z.array(z.string()),
});

// 2. Define the TanStack AI tool (for the LLM)
const showResultsDef = toolDefinition({
  name: 'show_results',
  description: 'Display a results table',
  inputSchema: showResultsInput,
  outputSchema: z.object({ displayed: z.boolean() }),
});

const showResults = showResultsDef.client(() => ({ displayed: true }));
const toolDefs = clientTools(showResults);

// 3. Define renderers — name and schema come from the tool definition
const toolRenderers = buildToolRenderers(
  toolRenderer(showResultsDef, ({ entityName, columns }) => (
    // entityName is string, columns is string[] — fully typed!
    <ResultsTable entity={entityName} columns={columns} />
  )),
);

// 4. Wire it up
const { messages, sendMessage, isLoading, stop } = useChat({
  connection,
  tools: toolDefs,
});

<AiChat
  messages={messages}
  isLoading={isLoading}
  onSendMessage={(text) => sendMessage(text)}
  onStop={stop}
  toolRenderers={toolRenderers}
/>
```

**How it works:**
- `toolRenderer(def, render)` extracts `name` and `inputSchema` from the tool definition — single source of truth
- The render function args are inferred from the definition's schema — full autocomplete, no `as` casts
- At runtime, `AiChatMessage` calls `schema.safeParse()` on the raw tool-call arguments before passing them to your render function
- `buildToolRenderers(...)` collects the entries into a `ToolRenderers` record

## Suggestion Buttons

Return a choices object as a tool result content to render interactive suggestion buttons. Buttons disappear after the user sends another message.

The choices format:
```json
{
  "type": "choices",
  "prompt": "How would you like to proceed?",
  "options": [
    { "id": "approve", "label": "Approve Document", "recommended": true },
    { "id": "reject", "label": "Reject Document" }
  ]
}
```

Handle selection explicitly or let the default behavior send `option.label` as a message:

```tsx
<AiChat
  messages={messages}
  isLoading={isLoading}
  onSendMessage={(text) => sendMessage(text)}
  onStop={stop}
  onChoiceSelect={(option) => {
    sendMessage(option.value ?? option.label);
  }}
/>
```

## Tool Grouping

Group consecutive tool-only assistant messages into expandable badges — useful when the LLM performs multiple tool calls before replying.

```tsx
<AiChat
  messages={messages}
  isLoading={isLoading}
  onSendMessage={(text) => sendMessage(text)}
  onStop={stop}
  enableToolGrouping
  toolDisplayNames={{
    search_database: 'Searching Database',
    filter_results: 'Filtering Results',
  }}
/>
```

- The latest group is expanded by default; older groups are collapsed
- Tools with an empty string display name are hidden from the badge

## Error Display

Pass an `Error` object to show an inline error banner:

```tsx
<AiChat
  messages={messages}
  isLoading={isLoading}
  onSendMessage={(text) => sendMessage(text)}
  onStop={stop}
  error={error}
/>
```

## API Reference

### `<AiChat>`

Main chat UI component. Works with TanStack AI's `UIMessage` format.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `messages` | `UIMessage[]` | required | Messages from `useChat` |
| `isLoading` | `boolean` | required | Loading state from `useChat` |
| `onSendMessage` | `(content: string) => void` | required | Send handler |
| `onStop` | `() => void` | required | Stop/abort handler |
| `onClearChat` | `() => void` | — | Clear handler |
| `onChoiceSelect` | `(option: ChoiceOption) => void` | — | Suggestion button handler (default: sends `option.label`) |
| `toolRenderers` | `ToolRenderers` | — | Map of tool names to `DisplayTool` instances (created via `toolRenderer()`) |
| `assistantName` | `string` | `"AI Assistant"` | Assistant display name |
| `title` | `string` | — | Chat title in the header |
| `emptyState` | `ReactNode` | — | Custom empty state |
| `placeholder` | `string` | — | Input placeholder |
| `showClearButton` | `boolean` | `true` | Show the clear button |
| `toolDisplayNames` | `Record<string, string>` | — | Human-readable tool names (empty string hides the tool) |
| `enableToolGrouping` | `boolean` | `false` | Group consecutive tool-only messages |
| `error` | `Error \| null` | — | Inline error banner |

### `AgentHubAdapterConfig`

Configuration for the AgentHub adapter.

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `baseUrl` | `string` | required | AgentHub base URL (`/chat/completions` is appended) |
| `model` | `{ vendor: 'openai' \| 'anthropic'; name: string }` | required | Model config |
| `accessToken` | `string \| () => string \| null` | required | Bearer token (refreshed per request if function) |
| `systemPrompt` | `string \| () => string` | — | System prompt prepended to messages (function form is called per request) |
| `maxTokens` | `number` | `2048` | Max response tokens |
| `temperature` | `number` | `0.7` | Sampling temperature |
| `tools` | `ReadonlyArray<AnyClientTool>` | — | Client tools — wire-format definitions are derived automatically |

### `toolRenderer(def, render)`

Create a named display tool renderer from a TanStack AI tool definition. Extracts the name and inputSchema — single source of truth.

```tsx
import { buildToolRenderers, toolRenderer } from '@/components/ui/ai-chat/types';

const toolRenderers = buildToolRenderers(
  toolRenderer(myToolDef, ({ name }) => <div>Hello {name}</div>),
  // name is typed from myToolDef.inputSchema — full autocomplete
);
```

### `buildToolRenderers(...entries)`

Collects `toolRenderer()` results into a `ToolRenderers` record.

```tsx
const renderers = buildToolRenderers(
  toolRenderer(toolA, renderA),
  toolRenderer(toolB, renderB),
);
```
