# AI Chat

A generic, reusable, and type-safe AI chat component for the Apollo Design System. Built with React, TypeScript, Tailwind CSS, and Zod, it supports OpenAI-compatible LLM APIs (OpenAI, Azure OpenAI, Ollama) with a pluggable `LLMProvider` interface, with an agentic tool loop, Zod-typed tool calling, markdown rendering, collapsible tool groups, suggestion buttons, and error display.

</div>

## Features

- **OpenAI-compatible API** â€” Works with OpenAI, Azure OpenAI, Ollama, and any compatible endpoint via `LLMProvider`
- **Agentic Tool Loop** â€” Automatically re-calls the LLM until no more tool calls remain
- **Zod Tool Calling** â€” Define tools with Zod schemas; input is validated and typed end-to-end
- **Markdown Rendering** â€” Renders assistant responses with GitHub Flavored Markdown
- **Collapsible Tool Groups** â€” Consecutive tool-only steps are grouped into expandable badges
- **Suggestion Buttons** â€” Interactive choice buttons rendered from tool results
- **Error Display** â€” Inline error banner for API and network errors
- **Message Injection** â€” `appendMessages` for programmatic message insertion
- **Session Storage** â€” Persist conversations to `sessionStorage` with a `storageKey`; omit for in-memory only
- **i18n Support** â€” Built-in internationalization via react-i18next
- **Accessible** â€” WCAG 2.1 compliant with keyboard navigation and ARIA live regions

## Quick Start

```tsx
import { AiChat } from '@/components/ui/ai-chat/components/ai-chat';
import { useAiChat } from '@/components/ui/ai-chat/hooks/use-ai-chat';
import { OpenAIChatProvider } from '@/components/ui/ai-chat/utils/ai-chat-openai-provider';

function BasicChat() {
  const chat = useAiChat({
    provider: new OpenAIChatProvider({
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
      systemPrompt: 'You are a helpful assistant.',
    }),
  });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      onClearChat={chat.clearChat}
      error={chat.error}
      title="AI Assistant"
    />
  );
}
```

## Tool Calling

There are two kinds of tools â€” **`ExecuteTool`** runs logic in the agentic loop and sends a result back to the LLM, **`DisplayTool`** renders UI and stops the loop (no result sent to the LLM).

Pass your tools to both `useAiChat` (for the agentic loop) and `<AiChat>` (for rendering). They share the same `Tools` map.

### ExecuteTool

Has a required `execute`. The agentic loop calls `execute`, sends the result back to the LLM, and continues until no more tool calls remain.

```tsx
import { z } from 'zod';
import { useAiChat } from '@/components/ui/ai-chat/hooks/use-ai-chat';
import { OpenAIChatProvider } from '@/components/ui/ai-chat/utils/ai-chat-openai-provider';
import type { Tools } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  search_records: {
    description: 'Search the database for matching records',
    inputSchema: z.object({
      query: z.string().describe('The search query'),
      limit: z.number().optional().describe('Max results to return'),
    }),
    execute: async ({ query, limit }) => {
      // args are fully typed from the Zod schema
      return await searchDatabase(query, limit);
    },
  },
};

function AgenticChat() {
  const chat = useAiChat({
    provider: new OpenAIChatProvider({
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
    }),
    tools,
    maxToolIterations: 10, // default
  });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      tools={tools}
      enableToolGrouping
      toolDisplayNames={{ search_records: 'Searching Records' }}
    />
  );
}
```

### DisplayTool

Has a required `render` and no `execute`. When the LLM calls a display tool, the loop stops immediately â€” no result is sent back â€” and the `render` function produces the UI.

```tsx
import type { DisplayTool, Tools } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  show_results_table: {
    description: 'Render a data table for the user',
    inputSchema: z.object({
      entityName: z.string(),
      columns: z.array(z.string()),
    }),
    render: (args) => (
      <ResultsTable
        entity={args.entityName}
        columns={args.columns}
      />
    ),
  } satisfies DisplayTool,
};

function DisplayChat() {
  const chat = useAiChat({
    provider: new OpenAIChatProvider({ baseUrl, model, accessToken }),
    tools,
  });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      tools={tools}
    />
  );
}
```

## Suggestion Buttons

Return a `ToolResultChoices` object from any tool's `execute` to render interactive choice buttons in the UI. Buttons disappear after the user sends another message.

```tsx
import type { ToolResultChoices } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  request_approval: {
    description: 'Ask the user how to proceed',
    inputSchema: z.object({}),
    execute: (): ToolResultChoices => ({
      type: 'choices',
      prompt: 'How would you like to proceed?',
      options: [
        { id: 'approve', label: 'Approve Document', recommended: true },
        { id: 'reject', label: 'Reject Document' },
        { id: 'request', label: 'Request Changes' },
      ],
    }),
  },
};

// Optionally handle the selection explicitly (default: sends option.label as a message)
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  onChoiceSelect={(option) => {
    chat.sendMessage(option.value ?? option.label);
  }}
/>
```

## Tool Grouping

Group consecutive tool-only assistant messages into expandable badges â€” useful when the agentic loop performs multiple steps before replying.

```tsx
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  enableToolGrouping
  toolDisplayNames={{
    search_database: 'Searching Database',
    filter_results: 'Filtering Results',
    generate_report: 'Generating Report',
  }}
/>
```

- The latest group is expanded by default; older groups are collapsed
- Tools with an empty string display name are hidden from the badge

## Error Display

Pass an `Error` object to show an inline error banner above the input:

```tsx
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  error={chat.error}  // set automatically by useAiChat on API errors
/>
```

You can also pass a custom error (e.g. from your own validation):

```tsx
const [customError, setCustomError] = useState<Error | null>(null);

<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  error={customError}
/>
```

## Message Injection (`appendMessages`)

Append messages directly to the chat history without making an LLM request â€” useful for mock playback, streaming integrations, or preloading historical context.

```tsx
const chat = useAiChat({ provider });

chat.appendMessages([
  {
    id: crypto.randomUUID(),
    role: 'assistant',
    content: [{ type: 'text', text: 'Here are your results.' }],
    timestamp: Date.now(),
  },
]);
```

## Dynamic System Prompt

Pass `systemPrompt` as a function in the provider config â€” it is called on every LLM request so it always reflects current state:

```tsx
function DynamicChat({ currentUser, currentContext }) {
  const chat = useAiChat({
    provider: new OpenAIChatProvider({
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
      systemPrompt: () =>
        `You are assisting ${currentUser.name}. Context: ${currentContext}.`,
    }),
  });
}
```

## Per-Context Storage

Pass a dynamic `storageKey` to persist history in `sessionStorage` and automatically reset when the context changes (e.g. switching between records):

```tsx
const chat = useAiChat({
  provider: new OpenAIChatProvider({ ... }),
  storageKey: `chat_${recordId}`,  // resets automatically when recordId changes
});
```

Omit `storageKey` entirely to use in-memory storage (history is lost on unmount/refresh).

## API Reference

### `useAiChat(options)`

Main hook for AI chat functionality.

**Options:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `provider` | `LLMProvider` | required | LLM provider instance (`OpenAIChatProvider` or any custom `LLMProvider`) |
| `tools` | `Tools \| () => Tools` | â€” | Tool definitions; if a function, evaluated once per render to produce the resolved tools |
| `maxToolIterations` | `number` | `10` | Max agentic loop iterations before throwing |
| `storageKey` | `string` | â€” | `sessionStorage` key for history persistence; omit for in-memory only |
| `onError` | `(error: Error) => void` | â€” | Called when a request fails |

**Returns:**

| Property | Type | Description |
|----------|------|-------------|
| `messages` | `ChatMessage[]` | Full conversation history |
| `isLoading` | `boolean` | True while waiting for LLM response |
| `error` | `Error \| null` | Last API error (cleared on next send) |
| `tools` | `Tools \| undefined` | Resolved tools â€” pass directly to `<AiChat tools={chat.tools}>` |
| `sendMessage` | `(content: string) => Promise<void>` | Send a user message |
| `stop` | `() => void` | Abort the in-flight request |
| `clearChat` | `() => void` | Clear history and storage |
| `appendMessages` | `(msgs: ChatMessage[]) => void` | Inject messages without an LLM call |

### `<AiChat>`

Main chat UI component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `messages` | `ChatMessage[]` | required | Messages array |
| `isLoading` | `boolean` | required | Loading state |
| `onSendMessage` | `(content: string) => void` | required | Message handler |
| `onStop` | `() => void` | required | Stop handler |
| `onClearChat` | `() => void` | â€” | Clear handler |
| `onChoiceSelect` | `(option: ChoiceOption) => void` | â€” | Suggestion button handler (default: sends `option.label`) |
| `tools` | `Tools` | â€” | Tool map â€” calls `tool.render(args)` per display tool call |
| `assistantName` | `string` | `"AI Assistant"` | Assistant display name |
| `title` | `string` | â€” | Chat title shown in the header |
| `emptyState` | `ReactNode` | â€” | Custom empty state |
| `placeholder` | `string` | â€” | Input placeholder |
| `showClearButton` | `boolean` | `true` | Show the clear conversation button |
| `toolDisplayNames` | `Record<string, string>` | â€” | Human-readable names for tools (empty string hides the tool) |
| `enableToolGrouping` | `boolean` | `false` | Group consecutive tool-only messages into collapsible badges |
| `error` | `Error \| null` | â€” | Renders an inline error banner when set |

### Key Types

**ChatMessage** â€” Discriminated union by role

```tsx
type ChatMessage = UserMessage | AssistantMessage | ToolMessage;

interface UserMessage {
  role: 'user';
  id: string;
  content: TextPart[];          // always an array of parts
  timestamp: number;
}

interface AssistantMessage {
  role: 'assistant';
  id: string;
  content: Array<TextPart | ToolCallPart>;
  timestamp: number;
}

interface ToolMessage {
  role: 'tool';
  id: string;
  content: ToolResultPart[];
  timestamp: number;
}
```

**Content Parts** â€” Discriminated by `type`

```tsx
interface TextPart     { type: 'text';        text: string }
interface ToolCallPart { type: 'tool-call';   toolCallId: string; toolName: string; args: Record<string, unknown> }
interface ToolResultPart { type: 'tool-result'; toolCallId: string; toolName: string; result: unknown }
```

**Tool** â€” Union of `ExecuteTool` and `DisplayTool`; parametric on Zod schemas

```tsx
// Runs logic in the agentic loop; result sent back to the LLM
interface ExecuteTool<
  TInputSchema extends ZodType = ZodType,
  TOutputSchema extends ZodType = ZodType,
> {
  description: string;
  inputSchema: TInputSchema;
  outputSchema?: TOutputSchema;
  execute(args: z.infer<TInputSchema>): MaybePromise<z.infer<TOutputSchema>>;
}

// Renders UI and stops the agentic loop; no result sent to the LLM
interface DisplayTool<TInputSchema extends ZodType = ZodType> {
  description: string;
  inputSchema: TInputSchema;
  render(args: z.infer<TInputSchema>): ReactNode;
}

type Tools = Record<string, Tool>;
```

**LLMProvider** â€” Interface implemented by all providers

```tsx
interface LLMProvider {
  chat(
    messages: ChatMessage[],
    tools: Tools | undefined,
    signal: AbortSignal,
  ): Promise<AssistantMessage>;
}
```

**OpenAIChatProviderConfig** â€” OpenAI, Azure OpenAI, Ollama, and compatible endpoints

```tsx
interface OpenAIChatProviderConfig {
  baseUrl: string;
  model: string;
  apiVersion?: string;                         // appended as ?api-version= (Azure)
  accessToken: string | (() => string | null); // refreshed per request
  systemPrompt?: string | (() => string);
  toolChoice?: 'auto' | 'none' | 'required';
  maxTokens?: number;                          // default 2048
  temperature?: number;                        // default 0.7
  fallbackResponse?: string;
}
```

**ToolResultChoices** â€” Return this from `execute` to render suggestion buttons

```tsx
interface ToolResultChoices {
  type: 'choices';
  prompt: string;
  options: ChoiceOption[];
}

interface ChoiceOption {
  id: string;
  label: string;
  value?: string;               // sent as message if provided, otherwise label
  recommended?: boolean;        // highlights the option
}
```
