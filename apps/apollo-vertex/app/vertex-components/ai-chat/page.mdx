# AI Chat

A generic, reusable, and type-safe AI chat component for the Apollo Design System. Built with React, TypeScript, Tailwind CSS, and Zod, it supports OpenAI-compatible LLM APIs with features like an agentic tool loop, Zod-typed tool calling, markdown rendering, collapsible tool groups, suggestion buttons, and error display.

</div>

## Features

- **OpenAI-compatible API** — Works with OpenAI, Azure OpenAI, and any compatible endpoint
- **Agentic Tool Loop** — Automatically re-calls the LLM until no more tool calls remain
- **Zod Tool Calling** — Define tools with Zod schemas; input is validated and typed end-to-end
- **Markdown Rendering** — Renders assistant responses with GitHub Flavored Markdown
- **Collapsible Tool Groups** — Consecutive tool-only steps are grouped into expandable badges
- **Suggestion Buttons** — Interactive choice buttons rendered from tool results
- **Error Display** — Inline error banner for API and network errors
- **Message Injection** — `appendMessages` for programmatic message insertion
- **Persistent History** — Saves conversations to session or local storage; auto-resets on key change
- **i18n Support** — Built-in internationalization via react-i18next
- **Accessible** — WCAG 2.1 compliant with keyboard navigation and ARIA live regions

## Quick Start

```tsx
import { AiChat } from '@/components/ui/ai-chat/components/ai-chat';
import { useAiChat } from '@/components/ui/ai-chat/hooks/use-ai-chat';

function BasicChat() {
  const chat = useAiChat({
    connection: {
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
    },
    systemPrompt: 'You are a helpful assistant.',
  });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      onClearChat={chat.clearChat}
      error={chat.error}
      title="AI Assistant"
    />
  );
}
```

## Tool Calling

There are two kinds of tools — **`ExecuteTool`** runs logic in the agentic loop and sends a result back to the LLM, **`DisplayTool`** renders UI and stops the loop (no result sent to the LLM).

Pass your tools to both `useAiChat` (for the agentic loop) and `<AiChat>` (for rendering). They share the same `Tools` map.

### ExecuteTool

Has a required `execute` and an optional `render`. The agentic loop calls `execute`, sends the result back to the LLM, and continues until no more tool calls remain.

```tsx
import { z } from 'zod';
import { useAiChat } from '@/components/ui/ai-chat/hooks/use-ai-chat';
import type { Tools } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  search_records: {
    description: 'Search the database for matching records',
    inputSchema: z.object({
      query: z.string().describe('The search query'),
      limit: z.number().optional().describe('Max results to return'),
    }),
    execute: async ({ query, limit }) => {
      // args are fully typed from the Zod schema
      return await searchDatabase(query, limit);
    },
  },
};

function AgenticChat() {
  const chat = useAiChat({
    connection: {
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
    },
    tools,
    maxToolIterations: 10, // default
  });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      tools={tools}
      enableToolGrouping
      toolDisplayNames={{ search_records: 'Searching Records' }}
    />
  );
}
```

### DisplayTool

Has a required `render` and no `execute`. When the LLM calls a display tool, the loop stops immediately — no result is sent back — and the `render` function produces the UI.

```tsx
import type { DisplayTool, Tools } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  show_results_table: {
    description: 'Render a data table for the user',
    inputSchema: z.object({
      entityName: z.string(),
      columns: z.array(z.string()),
    }),
    render: (toolCall) => (
      <ResultsTable
        entity={toolCall.args.entityName as string}
        columns={toolCall.args.columns as string[]}
      />
    ),
  } satisfies DisplayTool,
};

function DisplayChat() {
  const chat = useAiChat({ connection, tools });

  return (
    <AiChat
      messages={chat.messages}
      isLoading={chat.isLoading}
      onSendMessage={chat.sendMessage}
      onStop={chat.stop}
      tools={tools}
    />
  );
}
```

## Suggestion Buttons

Return a `ToolResultChoices` object from any tool's `execute` to render interactive choice buttons in the UI. Buttons disappear after the user sends another message.

```tsx
import type { ToolResultChoices } from '@/components/ui/ai-chat/utils/ai-chat-types';

const tools: Tools = {
  request_approval: {
    description: 'Ask the user how to proceed',
    inputSchema: z.object({}),
    execute: (): ToolResultChoices => ({
      type: 'choices',
      prompt: 'How would you like to proceed?',
      options: [
        { id: 'approve', label: 'Approve Document', recommended: true },
        { id: 'reject', label: 'Reject Document' },
        { id: 'request', label: 'Request Changes' },
      ],
    }),
  },
};

// Optionally handle the selection explicitly (default: sends option.label as a message)
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  onChoiceSelect={(option) => {
    chat.sendMessage(option.value ?? option.label);
  }}
/>
```

## Tool Grouping

Group consecutive tool-only assistant messages into expandable badges — useful when the agentic loop performs multiple steps before replying.

```tsx
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  enableToolGrouping
  toolDisplayNames={{
    search_database: 'Searching Database',
    filter_results: 'Filtering Results',
    generate_report: 'Generating Report',
  }}
/>
```

- The latest group is expanded by default; older groups are collapsed
- Tools with an empty string display name are hidden from the badge

## Error Display

Pass an `Error` object to show an inline error banner above the input:

```tsx
<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  error={chat.error}  // set automatically by useAiChat on API errors
/>
```

You can also pass a custom error (e.g. from your own validation):

```tsx
const [customError, setCustomError] = useState<Error | null>(null);

<AiChat
  messages={chat.messages}
  isLoading={chat.isLoading}
  onSendMessage={chat.sendMessage}
  onStop={chat.stop}
  error={customError}
/>
```

## Message Injection (`appendMessages`)

Append messages directly to the chat history without making an LLM request — useful for mock playback, streaming integrations, or preloading historical context.

```tsx
const chat = useAiChat({ connection, systemPrompt });

chat.appendMessages([
  {
    id: crypto.randomUUID(),
    role: 'assistant',
    content: [{ type: 'text', text: 'Here are your results.' }],
    timestamp: Date.now(),
  },
]);
```

## Dynamic System Prompt

Pass `systemPrompt` as a function — it is called on every LLM request so it always reflects current state:

```tsx
function DynamicChat({ currentUser, currentContext }) {
  const chat = useAiChat({
    connection: {
      baseUrl: 'https://api.openai.com/v1',
      model: 'gpt-4o',
      accessToken: () => getApiToken(),
    },
    systemPrompt: () =>
      `You are assisting ${currentUser.name}. Context: ${currentContext}.`,
  });
}
```

## Per-Context Storage

Pass a dynamic `messagesKey` to automatically reset history when the context changes (e.g. switching between records):

```tsx
const chat = useAiChat({
  connection,
  storage: {
    type: 'local',                          // 'session' | 'local' | 'none'
    messagesKey: `chat_${recordId}`,        // resets automatically when recordId changes
  },
});
```

## API Reference

### `useAiChat(options)`

Main hook for AI chat functionality.

**Options:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `connection` | `ChatConnection` | required | Network and auth config |
| `systemPrompt` | `string \| () => string` | — | System prompt, re-evaluated per request when a function |
| `tools` | `Tools` | — | Tool definitions with Zod schemas and execute handlers |
| `toolChoice` | `"auto" \| "none" \| "required"` | `"auto"` | Tool selection strategy |
| `maxTokens` | `number` | `2048` | Max tokens per response |
| `temperature` | `number` | `0.7` | Sampling temperature |
| `maxToolIterations` | `number` | `10` | Max agentic loop iterations before throwing |
| `fallbackResponse` | `string` | — | Text used when the assistant returns an empty response |
| `storage` | `StorageConfig` | `{ type: "session" }` | Persistence configuration |
| `onError` | `(error: Error) => void` | — | Called when a request fails |

**Returns:**

| Property | Type | Description |
|----------|------|-------------|
| `messages` | `ChatMessage[]` | Full conversation history |
| `isLoading` | `boolean` | True while waiting for LLM response |
| `error` | `Error \| null` | Last API error (cleared on next send) |
| `sendMessage` | `(content: string) => Promise<void>` | Send a user message |
| `stop` | `() => void` | Abort the in-flight request |
| `clearChat` | `() => void` | Clear history and storage |
| `appendMessages` | `(msgs: ChatMessage[]) => void` | Inject messages without an LLM call |

### `<AiChat>`

Main chat UI component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `messages` | `ChatMessage[]` | required | Messages array |
| `isLoading` | `boolean` | required | Loading state |
| `onSendMessage` | `(content: string) => void` | required | Message handler |
| `onStop` | `() => void` | required | Stop handler |
| `onClearChat` | `() => void` | — | Clear handler |
| `onChoiceSelect` | `(option: ChoiceOption) => void` | — | Suggestion button handler (default: sends `option.label`) |
| `tools` | `Tools` | — | Tool map — calls `tool.render(toolCall)` per tool call part |
| `assistantName` | `string` | `"AI Assistant"` | Assistant display name |
| `title` | `string` | — | Chat title shown in the header |
| `emptyState` | `ReactNode` | — | Custom empty state |
| `placeholder` | `string` | — | Input placeholder |
| `showClearButton` | `boolean` | `true` | Show the clear conversation button |
| `toolDisplayNames` | `Record<string, string>` | — | Human-readable names for tools (empty string hides the tool) |
| `enableToolGrouping` | `boolean` | `false` | Group consecutive tool-only messages into collapsible badges |
| `error` | `Error \| null` | — | Renders an inline error banner when set |

### Key Types

**ChatMessage** — Discriminated union by role

```tsx
type ChatMessage = UserMessage | AssistantMessage | ToolMessage;

interface UserMessage {
  role: 'user';
  id: string;
  content: TextPart[];          // always an array of parts
  timestamp: number;
}

interface AssistantMessage {
  role: 'assistant';
  id: string;
  content: Array<TextPart | ToolCallPart>;
  timestamp: number;
}

interface ToolMessage {
  role: 'tool';
  id: string;
  content: ToolResultPart[];
  timestamp: number;
}
```

**Content Parts** — Discriminated by `type`

```tsx
interface TextPart     { type: 'text';        text: string }
interface ToolCallPart { type: 'tool-call';   toolCallId: string; toolName: string; args: Record<string, unknown> }
interface ToolResultPart { type: 'tool-result'; toolCallId: string; toolName: string; result: unknown }
```

**Tool** — Union of `ExecuteTool` and `DisplayTool`; parametric on Zod schemas

```tsx
// Runs logic in the agentic loop; result sent back to the LLM
interface ExecuteTool<
  TInputSchema extends ZodType = ZodType,
  TOutputSchema extends ZodType = ZodType,
> {
  description: string;
  inputSchema: TInputSchema;
  outputSchema?: TOutputSchema;
  execute(args: z.infer<TInputSchema>): MaybePromise<z.infer<TOutputSchema>>;
}

// Renders UI and stops the agentic loop; no result sent to the LLM
interface DisplayTool<TInputSchema extends ZodType = ZodType> {
  description: string;
  inputSchema: TInputSchema;
  render(toolCall: ToolCallPart): ReactNode;
}

type Tool<
  TInputSchema extends ZodType = ZodType,
  TOutputSchema extends ZodType = ZodType,
> = ExecuteTool<TInputSchema, TOutputSchema> | DisplayTool<TInputSchema>;

type Tools = Record<string, Tool>;
```

**ChatConnection** — Network and auth config

```tsx
interface ChatConnection {
  baseUrl: string;
  model: string;
  apiVersion?: string;          // appended as ?api-version= (Azure)
  accessToken: string | (() => string | null);  // refreshed per request
}
```

**ToolResultChoices** — Return this from `execute` to render suggestion buttons

```tsx
interface ToolResultChoices {
  type: 'choices';
  prompt: string;
  options: ChoiceOption[];
}

interface ChoiceOption {
  id: string;
  label: string;
  value?: string;               // sent as message if provided, otherwise label
  recommended?: boolean;        // highlights the option
}
```
