# Feedback Collection

Patterns for collecting and displaying user feedback on AI-generated content, integrated with the UiPath Platform Feedback API via [`@uipath/vs-feedback`](https://github.com/UiPath/vertical-solutions).

This page documents the two main flows:

1. **Submit** — Users vote on AI output (👍/👎) or override sections with a reason. Maps to `PlatformFeedbackClient.submit()`.
2. **Read** — A dashboard lists collected feedback with filters, KPIs, and expandable rows. Maps to `PlatformFeedbackClient.getByAgentId()`.

For the standalone vote control, see [Feedback Vote Widget](/patterns/feedback-vote-widget). Triage and severity classification are out of scope for this page — they're a separate workflow.

> The submit + read examples below assume a shared per-vertical `TCustom` and `TComment` shape. The library is generic over both — what's shown is one concrete pattern.

```tsx
// Per-vertical types — define once, reuse across submit and read.
type MyCustom = {
  sectionName: string;
  overrideReason?: string;
  aiProducedFields: {
    aiSeverity: "Critical" | "High" | "Medium" | "Low" | null;
    aiTriaged: boolean;
  };
};

type MyComment = {
  text: string;          // free-form user text
  feedbackToAgent?: string;
};
```

### Submit handler shape

```tsx
import type { FeedbackDraft } from "@uipath/vs-feedback";
import { client } from "./feedback-client"; // your PlatformFeedbackClient<MyCustom, MyComment>
import { resolveTraceContext } from "./trace-context";

async function handleOverrideSubmit(args: {
  overriddenContent: string;
  reason: OverrideReason;
  agentFeedback: string;
  sectionName: string;
  maestroInstanceId: string;
}) {
  const { traceId } = await resolveTraceContext(args.maestroInstanceId);

  const draft: FeedbackDraft<MyCustom, MyComment> = {
    traceId,
    spanId: "", // Feedback API auto-resolves the root span (LLMOPS-2310)
    isPositive: false,
    customFields: {
      sectionName: args.sectionName,
      overrideReason: args.reason,
      aiProducedFields: { aiSeverity: null, aiTriaged: false },
    },
    comment: {
      text: args.overriddenContent,
      feedbackToAgent: args.agentFeedback,
    },
  };

  await client.submit(draft);
}
```

The mapper passed to `PlatformFeedbackClient` is responsible for stringifying `customFields` and `comment` into the wire format — submit handlers stay typed end-to-end.

### Fetching from the Platform Feedback API

`@uipath/vs-feedback` exposes `client.getByAgentId(timeRange)`, returning `PlatformFeedbackRecord<TCustom, TComment>[]`. The client's configured `agentId` and `timeRange` (epoch milliseconds) are the only inputs — no per-call agentId is needed. Adapt to the table's row shape with a small mapper:

```tsx
import { useQuery } from "@tanstack/react-query";
import { client } from "./feedback-client";

const ONE_MONTH_MS = 30 * 24 * 60 * 60 * 1000;

function useFeedbackData() {
  return useQuery({
    queryKey: ["feedback"],
    queryFn: async () => {
      const records = await client.getByAgentId({
        startTime: Date.now() - ONE_MONTH_MS,
        endTime: Date.now(),
      });
      return records.map(adaptToFeedbackRow);
    },
    staleTime: 60_000, // matches the lib's internal cache TTL
  });
}

function adaptToFeedbackRow(
  record: PlatformFeedbackRecord<MyCustom, MyComment>,
): FeedbackRow {
  return {
    id: record.id,
    isPositive: record.isPositive,
    section: record.metadata.customFields.sectionName,
    user: record.userEmail ?? "",
    reason: record.metadata.customFields.overrideReason ?? "—",
    severity: record.metadata.customFields.aiProducedFields.aiSeverity,
    createdAt: record.createdAt,
    comment: record.comment.text,
    agentFeedback: record.comment.feedbackToAgent ?? "",
  };
}
```

The adapter is per-vertical — it's the seam between the lib's generic shape and your domain. Keep it small.

### When to reach for `@uipath/apollo-dashboarding` instead

For richer dashboarding (custom layouts, data adapters, time-range filters, distribution charts), use [Dashboards](/dashboards). The pattern above is the lightweight alternative for simple lists with a handful of KPIs.

## Trace context

Both flows depend on resolving the trace context for the agent execution being reviewed. Use `MaestroTraceResolver` and `SolutionVersionResolver` from `@uipath/vs-platform-services`:

```tsx
import {
  MaestroTraceResolver,
  SolutionVersionResolver,
} from "@uipath/vs-platform-services";

const traceResolver = new MaestroTraceResolver(traceConfig);
const versionResolver = new SolutionVersionResolver(versionConfig);

async function resolveTraceContext(maestroInstanceId: string) {
  // PIMS InstanceId → real OTEL trace id (the API needs the OTEL one).
  const traceId = await traceResolver.resolveOtelTraceId(maestroInstanceId);
  // Informational; returns null on failure rather than throwing.
  const solutionVersion = await versionResolver.resolveVersion("MySolution");
  return { traceId, solutionVersion: solutionVersion ?? "" };
}
```

The Feedback API auto-resolves the root span when `spanId` is `""` (LLMOPS-2310) — drafts pass `spanId: ""` and let the platform fill it in. See the `vs-platform-services` package for the full resolver API.
